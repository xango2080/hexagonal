Architecture Hexagonal

Cette architecture est aussi appelée Ports et Adaptateurs

Allow an application to equally be driven by users, programs, automated test or batch scripts, and to be developed and tested in isolation from its eventual run-time devices and databases.

As events arrive from the outside world at a port, a technology-specific adapter converts it into a usable procedure call or message and passes it to the application. The application is blissfully ignorant of the nature of the input device. When the application has something to send out, it sends it out through a port to an adapter, which creates the appropriate signals needed by the receiving technology (human or automated). The application has a semantically sound interaction with the adapters on all sides of it, without actually knowing the nature of the things on the other side of the adapters.

Dans quel but ?

  • la séparation stricte du code métier et technique
  • l’inversion de dépendance : le code technique dépend du code métier, il est au service de ce dernier, pas l’inverse
  • une application maintenable et testable

Comment ca fonctionne ?

Présentation :

Application Provider Interface Primary Port

Ils s'occupent des interactions avec l'exterieur en utilisant l'hexagone.

Elle adapte les informations recues en valeurs métier avant de les envoyer dans l'hexagone. Puis elle recupère le resultat et l'adapte pour l'exterieur.

Service Provider Interface Secondary Port

Ils sont utilisés par l'hexagone pour communiquer avec l'exterieur.

Elle adapte les informations recues depuis l'hexagone pour les envoyer vers l'exterieur. Puis elle recupère le resultat et l'adapte pour l'hexagone.

Pour & contre ?

Pour

  • Testabilité : le code est S.O.L.I.D e
  • Flexible : Les partenaires sont échangable facilement
  • Métier propre : le métier n'est pas encombré par la technique

Contre

  • Lourd : la création de beaucoup de class peut devenir lourd
  • Méconnu : une architecture peut connu et confu au début
  • Métier : distinguer le métier de la technique

Zone grise

  • Aucune annotation ni framework dans le métier
  • Pas adapter pour tous les projets
  • C'est du ports et adapter easy !
